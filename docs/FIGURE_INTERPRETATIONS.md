# Scientific Figure Interpretations

This document provides detailed interpretations of each plot generated by `plot_scientific_analysis.py`, explaining what they show and how to read them for scientific rigor.

---

## 1. Extraction Precision by Source (Skills / Knowledge)

**Filename:** `scientific_extraction_precision_skills.png`, `scientific_extraction_precision_knowledge.png`

### What It Shows
Bar chart of **precision** (fraction of correct extractions) for each source (BERT standalone, BERT+LLM, LLM), with **Wilson 95% confidence intervals** as error bars. A red dashed line at 0.5 marks the chance level. Green bars indicate precision **significantly better than chance** at α = 0.05; gray bars are not significant.

### Interpretation
- **Above chance (0.5):** The model extracts items correctly more often than random guessing.
- **Green bars:** Statistically significant improvement over chance (binomial test p &lt; 0.05).
- **Wilson CI:** Narrow intervals → more precise estimates; wide intervals → higher uncertainty (often due to small sample sizes).
- **Typical finding:** LLM-based and hybrid models usually outperform BERT standalone; BERT alone may not reach significance for knowledge extraction.

## 2. Effect Size: Odds Ratio by Source (Skills / Knowledge)

**Filename:** `scientific_odds_ratio_skills.png`, `scientific_odds_ratio_knowledge.png`

### What It Shows
Horizontal bar chart of the **odds ratio** vs chance (p₀ = 0.5). Odds ratio = p/(1−p) when p₀ = 0.5. A vertical dashed line at OR = 1 marks chance. Blue bars = better than chance; red bars = worse than chance. X-axis is logarithmic when values span a wide range.

### Interpretation
- **OR > 1:** Model is more likely to be correct than incorrect, relative to chance.
- **OR = 1:** No effect vs chance.
- **OR &lt; 1:** Model performs worse than chance.
- **Example:** OR ≈ 12 means the odds of correct extraction are about 12× higher than the odds of incorrect extraction (vs 1× at chance).
- **Typical finding:** LLM and BERT+LLM show high OR; BERT standalone may show OR &lt; 1 (worse than chance).

## 3. Binomial Test (Skills / Knowledge)

**Filename:** `scientific_binomial_test_skills.png`, `scientific_binomial_test_knowledge.png`

### What It Shows
The **binomial distribution under H₀** (precision = 0.5) for the observed sample size. Gray bars = probability of each possible count of correct extractions if the model were guessing. Red vertical line = observed count; blue dashed line = expected count under H₀ (n × 0.5).

### Interpretation
- **H₀:** precision = 0.5 (model performs at chance).
- **H₁:** precision > 0.5 (model is better than chance).
- **Observed far right of distribution:** Unlikely under H₀ → reject H₀; evidence that the model outperforms chance.
- **Observed near center:** Consistent with H₀; no significant evidence of improvement over chance.
- **Purpose:** Visualizes why we reject or retain the null hypothesis in the binomial test.

## 4. FDR Volcano: Skill Time Trends

**Filename:** `scientific_trend_volcano.png`

### What It Shows
Scatter plot of **slope** (change in job count per month) vs **−log₁₀(q-value)**. Each point is a skill. Green = Emerging (slope &gt; 0, q &lt; 0.05); Red = Declining (slope &lt; 0, q &lt; 0.05); Gray = Stable. A horizontal dashed line marks q = 0.05.

### Interpretation
- **Above q = 0.05 line:** FDR-significant trend (unlikely to be a false discovery).
- **Right of x = 0:** Increasing demand (emerging).
- **Left of x = 0:** Decreasing demand (declining).
- **All gray:** No skill passes FDR correction; observed trends are within random variation.
- **Purpose:** Controls false discoveries when testing many skills; only robust trends are labeled Emerging/Declining.

## 5. Sample Regression: Skill Frequency vs Time

**Filename:** `scientific_trend_regression_samples.png`

### What It Shows
Individual scatter plots for selected skills: **month index** (x) vs **job count** (y). Each plot includes observed points and a fitted linear regression line (dashed black) with slope shown. Green = emerging/positive slope; red = declining/negative slope. Only **high-quality skills** are included (stop words like "and", "detail" are filtered out).

### Interpretation
- **Slope:** Average change in job count per month (e.g., slope = 4 → ~4 more postings per month).
- **Positive slope:** Growing demand (e.g., "design", "analyze data").
- **Negative slope:** Declining demand (e.g., "automate processes").
- **Scatter:** Variability in monthly counts; regression captures the overall trend.
- **Purpose:** Illustrates the regression model used for trend detection; filtered to meaningful skills only.

## 6. Calibration Curves (Reliability Diagram)

**Filename:** `scientific_calibration_curve.png`

### What It Shows
Three subplots for `confidence_score`, `semantic_density`, and `context_agreement`. X-axis = mean predicted probability; Y-axis = observed frequency. Diagonal = perfect calibration. Blue line = model. Brier score in each title (lower = better calibration).

### Interpretation
- **On diagonal:** Predictions match reality (e.g., when the model predicts 0.8, the event occurs ~80% of the time).
- **Above diagonal:** Model is underconfident (predicts lower than observed).
- **Below diagonal:** Model is overconfident (predicts higher than observed).
- **Brier score:** 0 = perfect; 0.25 ≈ random; < 0.1 suggests good calibration.
- **Purpose:** Evaluates whether pipeline scores (e.g., confidence) can be trusted as probabilities.

## 7. Power Curve

**Filename:** `scientific_power_curve.png`

### What It Shows
**Power** (probability of correctly rejecting H₀ when H₁ is true) vs **sample size n**. Purple line = power curve; red dashed = target power 0.80; green dotted = n = 37 (minimum n for 80% power). Assumptions: H₀ p = 0.5, H₁ p = 0.7, α = 0.05, one-sided binomial test.

### Interpretation
- **Power 0.80:** 80% chance to detect a true precision of 0.7 when testing against chance (0.5).
- **n = 37:** Minimum gold set size for 80% power under these assumptions.
- **Larger n:** Higher power; gold set sizes of 100–150 yield power ≈ 0.99.
- **Purpose:** Justifies gold set size design (see RESEARCH_QUESTIONS.md, power analysis).

## 8. Future-Domain Mapping

**Filename:** `scientific_future_mapping.png`

### What It Shows
Left: **Top-1 accuracy** bar (agreement between pipeline best domain and expert label). Right: **Mapping margin** histogram (top1_similarity − top2_similarity). Red dashed line in left plot = target 0.60.

### Interpretation
- **Top-1 accuracy:** Fraction of items where the pipeline’s top-ranked domain matches the expert.
- **Mapping margin:** Higher values → more confident assignment; low margin → ambiguous (top two domains similar).
- **Purpose:** Validates embedding-based domain mapping against expert labels.

## 9. Weight Sensitivity

**Filename:** `scientific_weight_sensitivity.png`

### What It Shows
Bar chart of **Jaccard similarity** between each weight configuration’s top-20 recommendations and the baseline (0.4×demand, 0.3×trend, 0.3×future). Green dashed = robust threshold (J ≥ 0.6).

### Interpretation
- **Jaccard:** |A ∩ B| / |A ∪ B| (intersection over union); 1 = identical top-20; 0 = no overlap.
- **High Jaccard:** Top recommendations are stable when weights change.
- **Low Jaccard:** Rankings are sensitive to weight choices.
- **Purpose:** Assesses robustness of priority score to parameter changes.
